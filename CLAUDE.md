# Fitness Tracker — Claude Code Instructions

## Branch workflow

Always work on a feature branch, never commit directly to `main`.

Before starting any implementation:
1. Create a branch: `git checkout -b <short-kebab-description>`
2. Do all work on that branch
3. When done, push the branch and open a PR into `main` with `gh pr create`

Branch naming: use a short kebab-case description of the feature, e.g. `drag-drop-exercises`, `last-weight-display`, `rest-day-improvements`.

## Repository

- GitHub repo: `ekgage/ekgage.github.io`
- Deploys from `main` via GitHub Pages
- Target branch for PRs: `main`
